01.Discussing the scope of var, let, and const:

  • var :

    • in function: Variables declared with var are function-scoped, which means they are only accessible within the function in which they are defined.

    globaly:If a var variable is declared outside of any function, it becomes a global variable and is accessible throughout the entire script..

  • let:

    • Variables declared with let are block-scoped, which means they are limited to the block in which they are defined

    when it declare in a function then it can not access outside of the function

  • const :

    • Variables declared with const also have block-level scope, just like let.

    02. The use cases of null and undefined

  • null:

    • null is a value that represents the explicit absence of any object value or a value that has been intentionally set to indicate the absence of value or an empty state.

    It is often used as a placeholder when you want to indicate that a variable or property has no meaningful value.

    we can use null to check if a variable or property intentionally holds no value.

  • undefined :

    • Variables that are declared but not initialized are automatically assigned the value undefined

    Function parameters that are not provided with a value default to undefined

    When we try to access an object property that doesn't exist, it returns undefined

    REST api:REST API stands for Representational State Transfer Application Programming Interface

    It is a set of architectural principles and constraints used for designing networked applications and web services.

    Key principles and characteristics of RESTful APIs include:

  • REST APIs use standard HTTP methods (GET, POST, PUT, DELETE, etc.) to perform operations on resources.
  • Resources can have multiple representations, such as JSON, XML, HTML, or plain text
